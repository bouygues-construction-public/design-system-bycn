# Shadows

> Shadows should be consistent and subtle, with consideration given to the size, direction, and intensity of the light source. The use of shadows should be intentional and purposeful, serving to enhance the user experience rather than hinder it.

For more details, see our [Shadows' guideline](https://zeroheight.com/9b39bb2a0/p/99c217-shadows/b/30ca23)

## Using [CSS Custom Properties](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties)

If you don't know CSS Custom Properties, you can see basic usage documentation [here](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties#Basic_usage). All our CSS Custom Properties for colors are listed at "Swatches" column just below:

```
var(--mas-shadow_100);
var(--mas-shadow_200);
var(--mas-shadow_300);
var(--mas-shadow_400);
```

## Using [Tailwind CSS](https://tailwindcss.com) utility classes

You can use shadows with Tailwind CSS utility classes about [Box Shadow](https://tailwindcss.com/docs/box-shadow). Here is the list of shadows extented by BYCN IT:

<Source language="json" code={JSON.stringify(tailwind.theme.extend.boxShadow, null, 2)} />
